**การวิเคราะห์การทดสอบความเร็วอินเทอร์เน็ตในแต่ละจังหวัดของประเทศไทย**

พี่ปี 4 เจ้าของบทความ เอาข้อมูลการทดสอบความเร็วเน็ตจากระบบที่ชื่อ M-Lab มาวิเคราะห์ว่าแต่ละจังหวัดในไทยเน็ตเร็วแค่ไหน M-Lab คือระบบวัดเน็ตที่ทำงานอยู่เงียบๆ ในพื้นหลัง ต่างจาก Speedtest ที่ผู้ใช้กดเองโดยตั้งใจ ข้อมูลที่ได้เลยมีลักษณะต่างกัน

ข้อมูลที่ดึงมามีหลายสิบล้านแถว เลยต้องใช้เครื่องมือพิเศษอย่าง Dask ที่ช่วยให้ประมวลผลข้อมูลใหญ่ได้โดยไม่ต้องโหลดทั้งหมดเข้า RAM พร้อมกัน และใช้ Google BigQuery ซึ่งเป็นระบบ query ข้อมูลขนาดใหญ่ของ Google ในการดึงข้อมูลต้นทาง

การวิเคราะห์แยกเป็น Broadband (เน็ตบ้าน) กับ Cellular (เน็ตมือถือ) และแยก download/upload โดยคำนวณค่าเฉลี่ย ความคลาดเคลื่อนมาตรฐาน และช่วงความเชื่อมั่น รวมถึงทดสอบด้วย ANOVA เพื่อดูว่าความแตกต่างระหว่างภูมิภาคมีนัยสำคัญทางสถิติจริงไหม ซึ่งผลออกมาว่ามีทุกกลุ่ม

ผลที่ได้ไม่ค่อยแปลกใจ คือเน็ตบ้านเร็วและเสถียรกว่าเน็ตมือถือในแทบทุกจังหวัด กทม.และภาคกลางดีที่สุดทั้งสองประเภท รองลงมาคือภาคตะวันออกและภาคใต้ ส่วนภาคอีสานแย่สุด โดยเฉพาะเน็ตมือถือที่ห่างจากอันดับต้นๆ ค่อนข้างมาก

จุดที่น่าสนใจกว่าผลคือปัญหาที่เจอระหว่างทำ ข้อมูลของ M-Lab ไม่ได้บอกตรงๆ ว่าแต่ละครั้งที่วัดนั้นมาจากเน็ตบ้านหรือมือถือ พี่คนนี้เลยเอา IP address ของแต่ละการทดสอบไปถามผ่าน ip-api.com เพื่อดูว่า ISP คืออะไร แล้วค่อยแยกประเภทจากนั้น ซึ่งวิธีนี้พอใช้ได้แต่ไม่แม่นยำทั้งหมด เพราะ ISP บางเจ้าให้บริการทั้งสองแบบ

นอกจากนี้ยังเจอ outlier ที่อ่างทองซึ่งมีความเร็ว Broadband สูงผิดปกติมาก พอขุดดูโดยจัดกลุ่มตามพิกัดและ IP พบว่าคนที่ทดสอบใช้ Cloudflare เป็น ISP ซึ่ง Cloudflare จริงๆ เป็นบริษัท CDN และ DNS ไม่ใช่ผู้ให้บริการเน็ตบ้านทั่วไป บ่งชี้ว่าการทดสอบนั้นอาจมาจากระบบอัตโนมัติหรือ server ไม่ใช่ผู้ใช้จริง และนี่แค่ที่จับได้ ในข้อมูลจริงอาจมี noise แบบนี้ปนอยู่อีก

**Speedtest by Ookla — DataProteins**

บทความนี้ใช้ข้อมูลของ Ookla ซึ่งเป็นเจ้าของแอป Speedtest สีฟ้าที่หลายคนคุ้น Ookla จะเก็บพิกัด GPS ทุกครั้งที่มีคนกดวัด และแบ่งพื้นที่ออกเป็น tile ขนาดประมาณ 610x610 เมตร ทำให้รู้ค่อนข้างแม่นว่าแต่ละจุดในเมืองความเร็วเป็นยังไง ข้อมูลที่ใช้คือ Q1 ปี 2026 มีการทดสอบใน กทม. รวมกว่า 566,000 ครั้ง แบ่งเป็น Mobile 129,914 ครั้ง และ Fixed broadband 437,023 ครั้ง

สิ่งที่พบคือเน็ตมือถือกับเน็ตบ้านให้ผล ranking ตรงข้ามกันแทบสมบูรณ์ เขตที่เน็ตมือถือเร็วที่สุดอย่างพระนคร (232 Mbps) ปทุมวัน วัฒนา ซึ่งล้วนอยู่ใจกลางเมือง กลับเป็นเขตที่เน็ตบ้านช้าที่สุดใน กทม. โดยปทุมวันอยู่อันดับสุดท้ายที่ 276 Mbps และพระนครรองสุดท้าย ในทางกลับกัน หนองจอกที่เน็ตมือถือช้าอันดับสองรองจากหนองแขม กลับมีเน็ตบ้านเร็วที่สุดใน กทม. ที่ 424 Mbps

เหตุผลที่อธิบายคือตึกเก่าในใจกลางเมืองอายุ 30-50 ปีเดินสายไฟเบอร์ใหม่ยากและมักยังใช้สายเดิมหรือแชร์กันในตึก ส่วนหมู่บ้านและคอนโดใหม่ชานเมืองได้ไฟเบอร์ลงพร้อมตั้งแต่สร้าง และค่าย AIS 3BB True ยังแข่งกันอยู่เพราะตลาดยังไม่อิ่มตัว ประกอบกับคนทำงานออฟฟิศใจกลางเมืองหันมาใช้ 5G แทนเน็ตบ้านไปแล้ว เลยยิ่งไม่มีแรงจูงใจอัปเกรดโครงสร้างพื้นฐานในย่านเก่า
